STEP 24 - POSTGRESQL AND ALEMBIC

1. Install and start PostgreSQL 18 on macOS:

   brew install postgresql@18
   brew services start postgresql@18

2. Add PostgreSQL commands to the current terminal:

   export PATH="$(brew --prefix postgresql@18)/bin:$PATH"

3. Confirm the server is reachable:

   pg_isready

4. Create the application role:

   createuser --login policy_copilot

5. Set a password for the role:

   psql postgres -c      "ALTER ROLE policy_copilot WITH PASSWORD 'CHANGE_THIS_PASSWORD';"

6. Create the database:

   createdb --owner=policy_copilot policy_copilot

7. Replace the existing DATABASE_* lines in .env using
   step24_env_postgresql.txt. Use the same password in
   DATABASE_URL. Do not commit .env.

8. Install dependencies:

   source ../.venv/bin/activate
   pip install -r requirements.txt

9. Apply migrations:

   alembic upgrade head

10. Check the migration revision:

    alembic current

11. Run tests:

    python -m pytest tests/test_database_settings.py -v
    python -m pytest tests/test_database_migrations.py -v
    python -m pytest

12. Start the API:

    uvicorn app.api.main:app       --host 127.0.0.1       --port 8000       --reload

Useful commands:

    alembic history
    alembic current
    alembic upgrade head
    alembic downgrade -1

For future model changes:

    alembic revision --autogenerate       -m "describe the schema change"

Always review an autogenerated migration before running it.

Important:
- Alembic creates/changes schema. It does not copy existing
  rows from the old SQLite file into PostgreSQL.
- Keep DATABASE_AUTO_CREATE_SCHEMA=false for PostgreSQL.
- Keep SQLite only for isolated tests or temporary local use.
